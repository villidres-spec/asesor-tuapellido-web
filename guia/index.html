Sección 01

Introducción

Esta guía te lleva paso a paso, desde cero, hasta tener tres herramientas digitales funcionando en internet con tu marca, tu logo y tu número de WhatsApp.

No necesitas saber programar. Solo seguir cada paso en el orden que aparece.

Qué vas a crear

  • Simulador de PPR (OptiMaxx Plus): tus prospectos ven cuánto tendrían acumulado al retiro.
  • Cotizador de Seguro de Vida (OptiMaxx Protección): ven el precio por paquete según edad y suma asegurada.
  • Portal de Links: página central con tu marca e enlace ideal para el bio de Instagram.
  • Las tres herramientas capturan datos del prospecto en tu Google Sheets automáticamente.

Herramientas que usarás (todas gratuitas)

HerramientaPara qué sirve
GitHubDonde vive el código de tu sitio.
VercelPublica tu sitio automáticamente al actualizar GitHub.
Google Sheets + Apps ScriptRecibe los datos de tus prospectos.
Claude AIPersonaliza el código con tu marca, sin programar.
💡Todo es completamente gratuito. No hay cargos mensuales ni pagos ocultos.

Tiempo estimado

Primera vez: 2 a 3 horas. Actualizaciones posteriores: 5 minutos.

Cómo leer esta guía

  • Pasos numerados → acciones en orden.
  • 💡 Amarillo → consejos útiles.
  • ✅ Verde → confirmación de que algo funcionó.
  • ⚠️ Naranja → advertencias importantes.
  • 🔧 Rojo → correcciones a la guía original.
Sección 02

Crear tus cuentas

2.1 GitHub

  1. Ve a github.com y haz clic en Sign up (esquina superior derecha).
  2. Escribe tu correo, crea una contraseña y elige un nombre de usuario (sin espacios).
  3. Verifica tu correo con el enlace que te enviaron.
  4. Cuando pregunten el plan, selecciona Free.
✅Guarda tu usuario y contraseña de GitHub en un lugar seguro. Los usarás constantemente.

2.2 Vercel

  1. Ve a vercel.com y haz clic en Sign Up.
  2. Selecciona Continue with GitHub. Esto conecta ambas cuentas.
  3. Autoriza la conexión.
  4. Cuando pregunten el tipo de proyecto, selecciona Personal → Hobby (plan gratuito).
💡Cuando subas un archivo a GitHub, Vercel lo detecta y publica los cambios en menos de 60 segundos.

2.3 Google

Si ya tienes Gmail, ya tienes cuenta de Google. Si no: accounts.google.com

Sección 03

Configurar Google Sheets

Crearás dos hojas: una para el PPR y otra para el Seguro. Cada una necesita un Apps Script que recibe los datos del formulario.

⚠️Este es el paso más técnico. Sigue los pasos exactamente. Si algo sale mal, el formulario no guardará datos pero el sitio seguirá funcionando.

3.1 Hoja para el Simulador PPR

Crear la hoja

  1. Ve a sheets.google.com y crea una hoja nueva (+).
  2. Nómbrala: Leads PPR [Tu Nombre]
  3. En la fila 1 escribe estos títulos (uno por celda):
    CeldaTítulo
    A1Fecha
    B1Nombre
    C1Edad
    D1WhatsApp
    E1Email
    F1Aportación mensual
    G1Portafolio

Conectar Apps Script

  1. En Google Sheets: Extensiones → Apps Script.
  2. Borra todo el texto que aparece en el editor.
  3. Copia y pega exactamente este código: 
    function doPost(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const data = JSON.parse(e.postData.contents);
  sheet.appendRow([
    new Date(), data.nombre, data.edad,
    data.whatsapp, data.email,
    data.aportacion, data.portafolio
  ]);
  return ContentService
    .createTextOutput(JSON.stringify({result:"success"}))
    .setMimeType(ContentService.MimeType.JSON);
}
  4. Guarda 💾 con nombre: Leads PPR
  5. Clic en Implementar → Nueva implementación.
  6. Engranaje ⚙️ → selecciona Aplicación web.
  7. Configura: Descripción = v1 · Ejecutar como = Yo · Acceso = Cualquier persona.
  8. Clic en Implementar.
  9. Autoriza permisos y selecciona tu cuenta de Google.
    🔧Corrección: Google mostrará "Esta aplicación no está verificada". Es normal — eres tú autorizando tu propio código. Haz clic en Configuración avanzada → Ir a Leads PPR (no seguro) → acepta los permisos.
  10. Copia la URL que aparece (https://script.google.com/macros/s/...) y guárdala en un documento aparte.
💡Si pierdes la URL: Apps Script → Implementar → Administrar implementaciones.

3.2 Hoja para el Cotizador de Seguro

Mismo proceso con estos cambios:

Nombre: Leads Seguro Vida [Tu Nombre]

CeldaTítulo
A1Fecha
B1Nombre
C1Edad
D1Sexo
E1WhatsApp
F1Email
G1Suma Asegurada
H1Temporalidad
I1Paquete

Código del Apps Script (diferente — pega este, no el anterior):

function doPost(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const data = JSON.parse(e.postData.contents);
  sheet.appendRow([
    new Date(), data.nombre, data.edad, data.sexo,
    data.whatsapp, data.email, data.sumaAsegurada,
    data.temporalidad, data.paquete
  ]);
  return ContentService
    .createTextOutput(JSON.stringify({result:"success"}))
    .setMimeType(ContentService.MimeType.JSON);
}
✅Al terminar debes tener DOS URLs guardadas: una para el PPR y una para el seguro.
Sección 04

Crear tu repositorio en GitHub

4.1 Crear el repositorio

  1. Entra a github.com.
  2. En la barra lateral izquierda, bajo "Repositories", haz clic en el botón verde New (o usa el ícono + en la barra superior).
    🔧Corrección: La guía original decía "esquina superior izquierda". El botón New está en la barra lateral izquierda bajo tu lista de repositorios.
  3. Llena: Repository name = asesor-tuapellido-web (minúsculas, guiones) · Visibilidad = Public · activa Add a README file.
  4. Clic en Create repository.

4.2 Conectar con Vercel

  1. Ve a vercel.com → Add New Project.
  2. Importa tu repositorio → Deploy.
  3. Vercel te dará una URL: asesor-tuapellido-web.vercel.app

Sección 06

Personalizar con Claude AI

6.1 Acceder a Claude

Ve a claude.ai, crea una cuenta gratuita e inicia sesión.

6.2 Darle el código a Claude

  1. Descarga el archivo base desde el repositorio de GitHub compartido (GENERICO_ppr_index.html o GENERICO_seguro_index.html).
    🔧Corrección: La guía no indicaba dónde están estos archivos. Los encuentras en el repositorio de GitHub compartido: clic en el archivo → botón Download raw file.
  2. En Claude, abre conversación nueva y escribe: "Voy a compartirte el código de mi simulador de PPR para personalizarlo."
  3. Adjunta el archivo con el ícono 📎.
  4. Dile exactamente qué cambiar.

6.3 Ejemplos de peticiones a Claude

  • "Cambia el WhatsApp a 521XXXXXXXXXX (código de país + 10 dígitos para México)."
    🔧Corrección: Los enlaces wa.me requieren código de país. Para México: 521 + 10 dígitos. Sin código de país, el botón de WhatsApp no funcionará.
  • "Reemplaza la URL de Google Sheets PPR con esta: [pega tu URL]."
  • "Agrega mi logo. Te adjunto el PNG."
  • "Cambia el nombre Asesor Landero por Asesor [Tu Apellido] en todos lados."
  • "Cambia el color de fondo a [color o código hex]."
  • "Actualiza las URLs del portal de links: simulador = [URL] · seguro = [URL]."
💡Siempre pídele al final: "Dame el archivo completo actualizado."

6.4 Obtener el archivo de Claude Corregido

🔧Corrección: La guía original decía "Claude mostrará un botón de descarga automáticamente". No siempre es así:

Opción A — Artefacto (panel derecho): aparece ícono ↓ en la esquina superior del panel. Descarga directo.

Opción B — Código en el chat: no hay botón de descarga. Haz esto:

  1. Copia el código del bloque (botón "Copiar").
  2. Abre Bloc de Notas (Windows) o TextEdit en modo texto plano (Mac).
  3. Pega y guarda como index.html — asegúrate que la extensión sea .html, no .txt.

6.5 Tasas mensuales

const ALLIANZ_MXN_RATE = 0.0672;  // Conservador en Pesos — 12 meses
const ALLIANZ_USD_RATE = 0.0333;  // Conservador Dolarizado — 12 meses
const UDIS_RATE        = 0.0847;  // AZ Revaluables — 12 meses

Divide el porcentaje entre 100. Ejemplo: 7.15% → 0.0715

Sección 07

Subir tus archivos y publicar

💡Cada vez que subas algo a GitHub, Vercel lo publica automáticamente en menos de 60 segundos.

7.1 Simulador PPR (raíz del repo)

  1. GitHub → tu repo → Add file → Upload files.
  2. Arrastra tu index.html personalizado.
  3. Commit: Agrego simulador PPR → Commit changes.
  4. Espera 30-60 s y verifica tu URL de Vercel.

7.2 Cotizador de Seguro (carpeta seguro/)

  1. Add file → Create new file.
  2. Nombre: seguro/index.html (la diagonal crea la carpeta).
  3. Pega el contenido de tu archivo de seguro personalizado.
  4. Commit: Agrego cotizador de seguro.

7.3 Portal de Links (carpeta links/)

  1. Add file → Create new file.
  2. Nombre: links/index.html
  3. Pega el contenido y commit: Agrego portal de links.

7.4 Logos

  1. Add file → Upload files → arrastra los PNG.
  2. El nombre del archivo debe coincidir exactamente con el del código.
  3. Commit: Agrego logos.
⚠️El nombre del logo es sensible a mayúsculas. LOGO.png ≠ logo.png. Deben coincidir exactamente.
Sección 08

Verificar que todo funciona

Verifica desde tu celular antes de compartir los enlaces.

8.1 Checklist Simulador PPR

  • Sitio carga sin errores en celular.
  • Logo aparece en la parte superior izquierda.
  • Nombre y especialidad correctos en el encabezado.
  • Llenar el formulario completo con datos de prueba.
  • Seleccionar portafolio → "Ver mi proyección" → resultados correctos.
  • Botón WhatsApp abre conversación contigo con mensaje pre-cargado.
  • Google Sheet PPR registra la fila de prueba.

8.2 Checklist Cotizador de Seguro

  • Sitio carga sin errores.
  • Logo de protección en el encabezado.
  • Llenar formulario → "Ver mi cotización" → tarjetas de paquetes con precios.
  • Al tocar el nombre de una cobertura se despliega la descripción.
  • "Me interesa este paquete" abre WhatsApp con paquete seleccionado.
  • Google Sheet Seguro registra la fila de prueba.

8.3 Checklist Portal de Links

  • Sitio carga sin errores.
  • Logo, nombre y especialidad al centro.
  • Botón Simulador PPR abre la herramienta correcta.
  • Botón Cotizador abre la herramienta correcta.
  • Botón WhatsApp e Instagram funcionan correctamente.
✅Si el logo no aparece, verifica que el nombre del PNG en GitHub coincida exactamente (mayúsculas incluidas) con el nombre en el código.

Sección 11

Preguntas frecuentes

¿Puedo cambiar colores y diseño?

Sí. Dile a Claude el color o compártele los códigos hex de tu paleta.

¿Por qué el logo no aparece?

El nombre del archivo no coincide con el del código. Verifica mayúsculas, guiones y extensión .png.

¿Puedo usar dominio propio?

Sí. Vercel → tu proyecto → Settings → Domains. Pídele ayuda a Claude con la configuración DNS.

¿Los datos de prospectos están seguros?

Van directo a tu Google Sheet privada. Solo tú tienes acceso. El sitio incluye Aviso de Privacidad (LFPDPPP).

¿Puedo actualizar el contenido después?

Sí. Claude modifica el archivo → descargas → subes a GitHub → Vercel actualiza en <60 s.

¿Qué hago si algo deja de funcionar?

Comparte el archivo con Claude y describe el problema. La mayoría tiene causa simple (nombre de archivo, URL de Apps Script, o código funcional tocado).

¿Puedo agregar más herramientas?

Sí. Crear archivo → subir en carpeta nueva en GitHub → nueva URL → agregarla al portal de links.

Nota final

Esta guía y sus archivos son una herramienta construida con tiempo y criterio. El resultado depende de quien la implemente.
